# Reglas del proyecto Alcancemos Web

- Mantener una identidad visual tecnológica, moderna, premium, clara y con mucho espacio negativo.
- Usar la paleta, variables CSS y estilo de componentes definidos en la base del proyecto.
- Crear componentes reutilizables, pequeños y semánticos antes de duplicar patrones visuales.
- Mantener TypeScript estricto y no usar `any`.
- Diseñar mobile-first, con accesibilidad y navegación por teclado.
- No inventar contenido comercial: casos de éxito, testimonios, métricas, clientes o resultados deben venir de una fuente aprobada.
- No copiar diseño, código, textos, secciones ni recursos visuales de Weplash; solo usarlo como referencia general de calidad y composición.
- Ejecutar `npm run lint` y `npm run build` antes de finalizar cada tarea.
- No reemplazar componentes funcionales sin revisar sus dependencias y usos existentes.
